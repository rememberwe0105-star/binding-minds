# 🧠 Project Context & Memory (핵심 기억 저장소)

> **이 문서는 향후 어떤 AI 어시스턴트(또는 개발자)가 프로젝트에 투입되더라도 절대 잊지 말아야 할 프로젝트의 핵심 전제조건과 아키텍처 원칙을 기록한 문서입니다.**

## 1. 아키텍처 원칙 (Architecture Principles)
- **프론트엔드-백엔드 완벽 분리 (Decoupling)**
  - 본 프로젝트는 프론트엔드(Next.js)와 백엔드(별도 API 서버)가 **완전히 분리되어 작업**되고 있습니다.
  - 프론트엔드 코드베이스 안에서 DB에 직접 접근하거나 서버사이드 비즈니스 로직을 자체적으로 처리해서는 안 됩니다.

## 2. 보안 및 인증 (Security & Authentication)
- **보안의 주체는 "백엔드"**
  - **Firebase** (사용자 인증, 토큰 발급, 사용자 Role 권한 관리)
  - **Stripe** (결제 처리, Secret Key 관리, 결제 인텐트 검증)
  - 위와 같이 민감한 보안, 결제, 권한 제어와 관련된 **모든 핵심 내용은 프론트엔드가 아닌 "백엔드 팀"에서 독점적으로 관리하고 제어**합니다.
  - 프론트엔드는 오직 백엔드로부터 받은 토큰이나 Publishable Key만을 사용하여 화면을 구성하고 렌더링하는 역할만 수행합니다.

## 3. 서버 환경 (Server Infrastructure)
- **사내 로컬 서버 사용**
  - 본 프로젝트의 백엔드 서버는 외부 클라우드(AWS 등)가 아닌 **회사 내부의 로컬 서버**를 사용하고 있습니다.
  - 현재 백엔드 API 서버 주소: `http://libertron.iptime.org:8787`
  - 프론트엔드의 `NEXT_PUBLIC_API_BASE_URL` 환경 변수는 항상 이 내부 서버 주소를 바라봐야 합니다.
