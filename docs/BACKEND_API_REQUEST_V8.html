P0 1. 수수료 모델 전환 — "기부자는 선택한 금액만 낸다"

왜 바꾸나요?

기부자 설문에서 "결제 시점에 수수료·세금 환급 얘기가 나오면 기부하는 기분이 떨어진다"는 피드백이 많았습니다. 그래서 기부자 화면에서 수수료 관련 항목을 전부 없애고, 플랫폼 운영비는 단체 정산액에서 차감하는 업계 표준 방식(application fee)으로 바꿉니다.

무엇이 어떻게 바뀌나요?

P0 2. 최소 기부금액 $10 → $5

NZ donation tax credit의 최소 기준액($5)에 맞추기 위한 변경입니다. 프론트는 이미 $5로 배포됐습니다.

  • POST /api/v1/checkout/donations 서버 검증을 amount >= 5로 완화
  • 관리자 설정 기본값 min_donation_minor: 500은 현재 값 그대로 유지하면 됩니다

P1 3. 익명 기부 (Donate Anonymously)

왜 필요한가요?

"내 기부가 공개 피드에 뜨는 게 부담스럽다"는 의견이 많았습니다. 결제 모달에 익명 체크박스를 추가했고, 프론트는 이미 아래 필드를 전송하고 있습니다.

POST /api/v1/checkout/donations
{
  "amount": 50,
  "currency": "NZD",
  "charityAccountId": "acct_xxx",
  "charityName": "Trees That Count",
  "addSupport": false,
  "recurring": false,
  "anonymous": true        // ← 신규
}

부탁드리는 작업

  • donations 테이블에 is_anonymous BOOLEAN 저장
  • 공개 노출 규칙 적용 (아래 표 참고)
노출 위치익명 기부 처리
공개 피드 / 실시간 알림 / 서포터 목록이름 비노출 — "Someone donated $50" 형태
단체 대시보드이름 비노출 (금액·시각만 표시)
본인 대시보드 (GET /me/donations)정상 표시 (본인은 자기 기록을 볼 수 있어야 함)
영수증 / 세금 요약정상 발급 (IRD 제출용이므로 실명 필요)

P1 4. 회사(법인) 기부 — 회사 명의 영수증

왜 필요한가요?

회사 차원에서 기부하고 싶다는 문의가 꾸준히 있었습니다. 핵심 요구사항은 "영수증이 개인 이름이 아니라 정식 회사 이름으로 발급되어야 한다"입니다. 또한 회사 기부는 개인의 33.33% donation tax credit이 아니라 법인 소득공제(tax deduction) 대상이라서, 영수증과 세금 안내 문구도 달라져야 합니다.

프론트 결제 모달에 Personal / As an Organisation 선택을 추가했고, 이미 아래 필드를 전송하고 있습니다.

POST /api/v1/checkout/donations
{
  "amount": 500,
  "currency": "NZD",
  "charityAccountId": "acct_xxx",
  "donorType": "organization",        // ← 신규: "individual" | "organization"
  "organizationName": "OSOPRO Ltd"    // ← 신규: 영수증 발급 명의
}

부탁드리는 작업

  • donations 테이블에 donor_type, organization_name 저장
  • 영수증(이메일 + PDF)의 수취인 명의를 organization_name으로 발급
  • 회사 기부 영수증에서는 33.33% tax credit 안내 문구 제외 (tax deduction 안내로 대체하거나 세금 문구 생략)
  • GET /me/donations 응답 항목에 donor_type, organization_name 필드 추가 — 프론트 대시보드에 개인/회사 필터 UI가 이미 구현되어 있고, 이 필드가 오는 순간 바로 동작합니다
  • 연간 세금 요약(tax summary) 계산에서 회사 기부 금액은 제외 (개인 tax credit 대상이 아니므로)
✅ 필드가 없어도 깨지지 않습니다
프론트는 donor_type이 없는 기존 기부 건을 전부 "개인 기부"로 간주하도록 처리해 두었습니다. DB 마이그레이션 시 기존 레코드는 NULL로 두셔도 됩니다.

P1 5. 게스트(비로그인) 기부 — v8.1 신규

왜 필요한가요?

기부 시작 단계에서 회원가입/로그인을 강제하면 이탈이 큽니다. 기부 완료율을 높이기 위해 로그인 없이 이름과 이메일만으로 기부를 완료할 수 있게 하고, 완료 후에 계정 생성을 유도하는 흐름으로 바꿉니다. 프론트에는 비로그인 시 이름/이메일 입력 UI와 완료 페이지의 "Create a free account" 유도 카드가 이미 배포되어 있습니다.

요청 엔드포인트 (신규)

POST /api/v1/checkout/donations/guest     ← 인증 헤더 없음

{
  "amount": 50,
  "currency": "NZD",
  "charityAccountId": "acct_xxx",
  "charityName": "Trees That Count",
  "recurring": false,
  "anonymous": false,
  "donorType": "individual",
  "guestName": "Aroha Ngata",          // ← 신규
  "guestEmail": "aroha@email.com"      // ← 신규
}

응답: 기존과 동일 { "sessionId": "...", "url": "https://checkout.stripe.com/..." }

부탁드리는 작업

  • 인증 없이 위 요청을 받아 Stripe Checkout Session 생성 (customer_email에 guestEmail 지정)
  • 이름+이메일 기반 고객 기록(guest customer) 생성 — 같은 이메일로 재기부 시 동일 고객으로 연결
  • 결제 완료 이메일(영수증)에 계정 연결 링크 포함 — 링크로 가입/로그인하면 게스트 기부 내역이 그 계정에 자동으로 연결되도록 (Firebase email link 인증 활용 제안)
  • 스팸/남용 방지: rate limit (IP당 분당 수 회) + 이메일 형식 검증
  • 정기 기부(recurring)는 게스트에게 허용할지 협의 필요 — 구독 관리를 위해 계정이 필요하므로 초기에는 게스트 정기 기부 차단(400 응답) 권장, 프론트가 에러 메시지 표시
✅ 기존 로그인 흐름은 그대로
로그인 사용자는 기존 POST /api/v1/checkout/donations(인증 필수)를 계속 사용합니다. 게스트 엔드포인트가 준비되기 전까지 비로그인 기부 시도는 프론트에서 에러 안내로 처리됩니다.

P2 6. GST 등록 상태 enum 변경

단체 온보딩 폼에서 "GST Exempt"가 실제 상황과 맞지 않는 경우가 많아(등록 여부를 모르는 담당자가 대부분), 더 현실적인 선택지로 바꿨습니다. POST /api/v1/charities/apply의 gst_registered 값입니다.

이전현재화면 표기
yesyes (유지)GST Registered
nono (유지)Not GST Registered
exempt삭제—
—not_sure (신규, 기본값)Not sure / To be confirmed
—in_progress (신규)Registration in progress
  • enum 검증에 not_sure, in_progress 추가, exempt 신규 입력 차단
  • 기존 exempt 데이터는 not_sure로 마이그레이션 권장
  • not_sure / in_progress는 승인 차단 사유가 아니며, 승인 후 팀에서 후속 확인하는 흐름입니다

P3 8. 금액별 기부 티어 API (유료 플랜 기능)

유료 플랜 단체가 "금액 프리셋별 설명(+사진)"을 직접 설정하는 기능입니다. 예: $25 → "묘목 5그루를 심습니다". 프론트 결제 모달은 티어 데이터가 있으면 자동으로 티어 카드 UI로 전환되도록 이미 구현되어 있습니다 (지금은 로컬 데모 데이터로 시연 중).

MethodEndpoint설명
GET/api/v1/charities/{id}/donation-tiers공개 조회 (누구나)
PUT/api/v1/me/charity/donation-tiers단체 관리자 설정 (유료 플랜만, 무료 플랜은 403 + 업그레이드 안내)
 
{
  "tiers": [
    { "amount": 25, "title": "Plant 5 native seedlings",
      "description": "Five natives planted and tracked.", "imageUrl": "https://..." }
  ]
}

이미지는 기존 이미지 업로드 API를 재사용하면 됩니다. 티어는 최대 5개 정도로 제한하는 것을 제안드립니다.

P3 9. Stripe Direct Debit (준비되면)

카드 외에 direct debit으로도 결제하고 싶다는 요청입니다. 급하지 않으니 Stripe 쪽 준비가 되면 진행해 주세요.

  • Stripe Checkout payment_method_types에 NZ direct debit 계열 추가 검토
  • Direct debit은 정산이 수일 지연되므로 donation_status에 processing 상태가 필요한지 확인
  • 정기 기부(subscription)와의 호환성 확인

프론트는 Stripe Checkout 리다이렉트 방식이라 백엔드 세션 설정만으로 바로 노출됩니다 — 프론트 추가 작업이 없습니다.
