P1 1. 게스트(비로그인) 기부

왜 필요한가요?

기부 시작 단계에서 회원가입/로그인을 강제하면 이탈이 큽니다. 기부 완료율을 높이기 위해 로그인 없이 이름과 이메일만으로 기부를 완료할 수 있게 하고, 완료 후에 계정 생성을 유도하는 흐름으로 바꿉니다. 프론트에는 비로그인 시 이름/이메일 입력 UI와 완료 페이지의 "Create a free account" 유도 카드가 이미 배포되어 있습니다.

요청 엔드포인트 (신규)

POST /api/v1/checkout/donations/guest     ← 인증 헤더 없음

{
  "amount": 50,
  "currency": "NZD",
  "charityAccountId": "acct_xxx",
  "charityName": "Trees That Count",
  "recurring": false,
  "anonymous": false,
  "donorType": "individual",
  "guestName": "Aroha Ngata",          // ← 신규
  "guestEmail": "aroha@email.com"      // ← 신규
}

응답: 기존과 동일 { "sessionId": "...", "url": "https://checkout.stripe.com/..." }

부탁드리는 작업

  • 인증 없이 위 요청을 받아 Stripe Checkout Session 생성 (customer_email에 guestEmail 지정)
  • 이름+이메일 기반 고객 기록(guest customer) 생성 — 같은 이메일로 재기부 시 동일 고객으로 연결
  • 결제 완료 이메일(영수증)에 계정 연결 링크 포함 — 링크로 가입/로그인하면 게스트 기부 내역이 그 계정에 자동으로 연결되도록 (Firebase email link 인증 활용 제안)
  • 스팸/남용 방지: rate limit (IP당 분당 수 회) + 이메일 형식 검증
  • 정기 기부(recurring)는 게스트에게 허용할지 협의 필요 — 구독 관리를 위해 계정이 필요하므로 초기에는 게스트 정기 기부 차단(400 응답) 권장, 프론트가 에러 메시지 표시
✅ 기존 로그인 흐름은 그대로
로그인 사용자는 기존 POST /api/v1/checkout/donations(인증 필수)를 계속 사용합니다. 게스트 엔드포인트가 준비되기 전까지 비로그인 기부 시도는 프론트에서 에러 안내로 처리됩니다.

정정 2. 플랫폼 팁(addSupport) — 론치 단계에서 제거

v8.0 문서의 수수료 표에서 "자발적 플랫폼 팁 $2 (addSupport): 유지"라고 안내드렸는데, 피드백 반영 결과 론치 단계에서는 팁 옵션도 노출하지 않기로 확정됐습니다. 기부자는 문자 그대로 "선택한 금액만" 결제합니다.

  • 프론트는 addSupport 파라미터를 더 이상 전송하지 않습니다 — 값이 와도 무시 처리만 해주세요 (하위 호환)
  • Checkout 금액 = amount 단독 (v8.0의 "+ 팁 $2" 부분은 삭제)
  • 팁 관련 DB 필드/정산 로직을 이미 만드셨다면 비활성화 (추후 재도입 시 별도 요청드리겠습니다)

참고 3. 백엔드 작업이 필요 없는 변경 (이미 배포됨)

변경내용
금액 동기화 버그 수정상세 페이지에서 고른 금액이 결제 모달에 그대로 반영되도록 수정
데모 charity 플랜 분리데모 로그인이 Free/Premium 2종으로 분리 — 요금제별 기능 차이(Analytics·Donor Updates 잠금) 시연 가능. 프론트 mock 기반이라 백엔드 무관이지만, 추후 실제 plan 필드(v8.0 P0-1) 연동 시 동일한 UI가 그대로 사용됩니다
게스트 기부 UI비로그인 시 이름/이메일 입력 + 완료 페이지 계정 생성 유도 카드 — 위 P1 엔드포인트만 열리면 즉시 동작

자주 나올 질문 (FAQ)

질문답변
게스트 기부자가 나중에 가입하면 내역이 연결되나요?네, 그게 핵심입니다. 이메일 기준으로 게스트 고객 기록을 계정에 병합해 주세요.
게스트 기부도 익명(anonymous) 가능한가요?네. 익명은 "공개 노출"만 막는 것이라 게스트와 독립적으로 동작합니다.
addSupport를 보내는 옛 요청이 오면?무시하면 됩니다. 프론트는 v8.1부터 보내지 않습니다.
v8.0 문서 내용 중 바뀐 건 팁뿐인가요?네. 나머지 v8.0 항목(수수료 모델, 최소 $5, 익명, 회사 기부, GST, 카테고리, 티어, direct debit)은 전부 그대로 유효합니다.

진행 제안: v8.0의 P0 2건(수수료 모델 + 최소 $5)을 먼저 진행하시고, 게스트 엔드포인트는 그다음 P1 묶음(익명·회사 기부)과 함께 진행해 주시면 프론트에서 순서대로 E2E 테스트하겠습니다.

문서가 자주 나가서 죄송합니다 — 피드백 반영 속도를 우선하다 보니 그렇습니다. 궁금한 점은 언제든 연락 주세요. 감사합니다! — Poma (프론트엔드)
